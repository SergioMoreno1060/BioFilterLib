02. INICIO RÁPIDO

Primeros pasos con BioFilterLib

1

Instalar la Librería

Descarga el repositorio y colócalo en tu carpeta de librerías de Arduino, luego incluye el header principal.

BioFilterLib.h 
#include <BioFilterLib.h>
2

Crear un Filtro FIR

Define los coeficientes (genera con Python/scipy o MATLAB) e instancia el filtro con el número de taps y el tamaño de bloque.

FIR_ECG.ino 
// Coeficientes pasa-bajas 40 Hz, Fs=960 Hz, 51 taps
const float32_t coeffs[51] = {
  -0.000722f, -0.001267f, /* ... 47 más ... */
   0.000722f
};

// Crear filtro: (coeficientes, numTaps, blockSize)
FIRFilter filtroECG(coeffs, 51, 1);
3

Filtrar en Tiempo Real

Llama a processSample() en el loop de Arduino para filtrado muestra a muestra con latencia mínima.

loop.ino 
void loop() {
  // Leer ADC y normalizar a [-1, 1]
  float32_t muestra = (analogRead(A0) - 2048.0f) / 2048.0f;

  // Aplicar filtro FIR
  float32_t filtrada = filtroECG.processSample(muestra);

  Serial.println(filtrada);
  // ~1.04 ms @ 960 Hz
  delayMicroseconds(1042);
}
03. ARQUITECTURA

Herencia desde CMSIS-DSP

BioFilterLib no reemplaza a CMSIS-DSP: lo encapsula. Cada clase C++ contiene una instancia de la estructura CMSIS correspondiente como miembro privado, exponiendo una API amigable para Arduino.

info

Composición vs. Herencia

BioFilterLib usa composición: cada clase C++ tiene como miembro privado la estructura CMSIS-DSP correspondiente. Esto expone la API de Arduino sin sacrificar las optimizaciones ARM nativas. WaveletFilter es especial: compone 4 instancias de FIRFilter para implementar el banco de filtros de análisis y síntesis Daubechies-4.
