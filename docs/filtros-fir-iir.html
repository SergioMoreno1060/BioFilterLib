Filtros FIR e IIR

Los filtros FIR (Respuesta al Impulso Finita) e IIR (Infinita) son los pilares del procesamiento de señales digitales biomédicas. BioFilterLib los implementa como clases C++ que encapsulan las estructuras optimizadas de CMSIS-DSP para el ARM Cortex-M3.

FIRFilter — Respuesta al Impulso Finita

CMSIS-DSP SUBYACENTE
arm_fir_instance_f32

La clase FIRFilter contiene como miembro privado una instancia de arm_fir_instance_f32 e invoca arm_fir_f32() para el procesamiento. Los 4 archivos arm_fir_f32.c / arm_fir_init_f32.c están incluidos en src/utils/.

functions Fundamento Matemático

Un filtro FIR de orden $N$ calcula la convolución discreta entre la señal de entrada $x[n]$ y sus $N$ coeficientes $b[k]$ (también llamados "taps"):

$$y[n] = \sum_{k=0}^{N-1} b[k] \cdot x[n-k]$$
$y[n]$

Muestra de salida en el instante $n$

$b[k]$

Coeficientes del filtro (pesos de la convolución). Se calculan con scipy/MATLAB.

$x[n-k]$

Muestra de entrada retrasada $k$ instantes. El filtro necesita $N-1$ muestras pasadas.

check_circle
Siempre estable

No tiene realimentación, por lo que no puede volverse inestable independientemente de los coeficientes.

check_circle
Fase lineal

Con coeficientes simétricos, el retardo de grupo es constante en todas las frecuencias. Crítico para preservar la morfología del ECG/EEG.

warning
Mayor carga computacional

Para la misma selectividad, un FIR necesita más coeficientes que un IIR. En Arduino Due a 84 MHz es perfectamente viable para tasas de muestreo típicas.

terminal Constructor

FIRFilter.h 
FIRFilter(float32_t* coeffs, uint16_t numTaps, uint16_t blockSize);
PARÁMETRO TIPO DESCRIPCIÓN
coeffsfloat32_t*Array de coeficientes del filtro. Debe permanecer válido mientras exista el objeto.
numTapsuint16_tNúmero de coeficientes (= orden + 1). Típico para bioseñales: 11–101.
blockSizeuint16_tTamaño del bloque CMSIS-DSP. Usar 1 para tiempo real; potencias de 2 para batch.
Memoria requerida: El buffer de estado interno tiene tamaño numTaps + blockSize - 1 elementos float32_t (4 bytes c/u). Para 51 taps + blockSize=1: 204 bytes.

code Métodos

processSample — tiempo real 
float32_t processSample(float32_t input);
Procesa una muestra individual manteniendo el estado interno. Ideal para el loop() de Arduino. Retorna: la muestra filtrada.
processBuffer — por lotes 
void processBuffer(float32_t* inputArray, float32_t* outputArray, uint32_t length);
Procesa un bloque completo. Mayor throughput gracias a las instrucciones SIMD del Cortex-M3. Los arrays no deben solaparse en memoria.

lightbulb Ejemplo Completo — Filtro Pasa-Bajas ECG

terminal FIR_ECG_lowpass.ino 
// FIR Pasa-Bajas 40 Hz, Fs=960 Hz, 51 taps (Ventana Hamming)
// Generado con: scipy.signal.firwin(51, 40, fs=960, window='hamming')
#include <BioFilterLib.h>

const float32_t coeffs[51] = {
  -0.000722f, -0.001267f, -0.001540f, -0.000996f,  0.000738f,
   0.003838f,  0.008015f,  0.012478f,  0.015748f,  0.015829f,
   0.010561f,  0.000000f, -0.015052f, -0.031823f, -0.046489f,
  -0.054661f, -0.052049f, -0.035751f,  0.000000f,  0.049774f,
   0.108029f,  0.168259f,  0.221893f,  0.261089f,  0.280000f,
   /* ... simétrico ... */
  -0.000722f
};

FIRFilter filtroECG(coeffs, 51, 1); // blockSize=1 para tiempo real

void setup() {
  Serial.begin(115200);
  analogReadResolution(12); // 12 bits en Arduino Due
}

void loop() {
  // Leer ADC y normalizar a rango [-1, 1]
  float32_t raw = (analogRead(A0) - 2048.0f) / 2048.0f;

  // Aplicar filtro FIR
  float32_t filtrada = filtroECG.processSample(raw);

  // Compensar retardo de grupo: (numTaps-1)/2 = 25 muestras
  Serial.print(raw,   6);  Serial.print(",");
  Serial.println(filtrada, 6);

  delayMicroseconds(1042); // ~960 Hz
}
Retardo de Grupo (Group Delay): Un FIR simétrico introduce un retardo constante de $(N-1)/2$ muestras. Para 51 taps = 25 muestras = 26 ms a 960 Hz. Este retardo es constante en todas las frecuencias (fase lineal).

IIRFilter — Secciones Biquad en Cascada

CMSIS-DSP SUBYACENTE
arm_biquad_casd_df1_inst_f32

La clase IIRFilter encapsula una instancia de arm_biquad_casd_df1_inst_f32 e invoca arm_biquad_cascade_df1_f32(). Implementa la forma directa I (Direct Form 1) para máxima estabilidad numérica.

functions Fundamento Matemático — Sección Biquad

Cada sección biquad implementa un filtro de segundo orden con coeficientes de avance ($b$) y de realimentación ($a$):

$$y[n] = b_0 x[n] + b_1 x[n-1] + b_2 x[n-2] \;-\; a_1 y[n-1] - a_2 y[n-2]$$

Para un filtro de orden $2M$, se encadenan $M$ secciones biquad, donde la salida de cada sección es la entrada de la siguiente:

$$y_k[n] = b_{0,k}\,x_k[n] + b_{1,k}\,x_k[n-1] + b_{2,k}\,x_k[n-2] - a_{1,k}\,y_k[n-1] - a_{2,k}\,y_k[n-2]$$

donde $x_{k+1}[n] = y_k[n]$: la salida de la etapa $k$ alimenta la entrada de la etapa $k+1$.

data_array

Formato de Coeficientes para CMSIS-DSP

Cada sección biquad necesita 5 coeficientes en el orden: { b0, b1, b2, a1, a2 }. El coeficiente $a_0 = 1$ se omite. Para un filtro de 2 etapas, el array tiene 10 elementos.

formato_coeficientes.ino 
// Filtro Notch 60 Hz, Fs=960 Hz, Q=30 — 1 sección biquad
// scipy: b,a = signal.iirnotch(60, 30, 960)
// Convertir a SOS con signal.tf2sos(b, a)
const float32_t notch60Hz[5] = {
  0.99034f,  // b0
 -1.80193f,  // b1
  0.99034f,  // b2
  1.80193f,  // a1  (signo invertido respecto a scipy)
 -0.98068f   // a2  (signo invertido respecto a scipy)
};

⚠️ CMSIS-DSP usa los coeficientes $a$ con signo invertido respecto a la convención de scipy/MATLAB. Verificar al generar los coeficientes.

check_circle
Alta selectividad con bajo orden

Un filtro notch IIR de 2do orden logra la misma atenuación que un FIR de 200+ coeficientes. Ideal para eliminar 50/60 Hz de interferencia eléctrica.

check_circle
Menor uso de memoria y CPU

Buffer de estado: solo 4 × numStages elementos float32. Para 1 etapa = 16 bytes. Retardo de grupo mínimo (~0 muestras para notch).

warning
Fase no lineal

El retardo de grupo varía con la frecuencia. Puede distorsionar la morfología de señales como el complejo QRS del ECG. Para análisis morfológico, preferir FIR.

lightbulb Ejemplo Completo — Filtro Notch 60 Hz ECG

terminal IIR_Notch_60Hz.ino 
// IIR Notch 60 Hz, Fs=960 Hz, Q=30 (1 etapa biquad)
// Generado con scipy.signal.iirnotch(60, 30, fs=960)
// Nota: CMSIS-DSP invierte el signo de a1 y a2 vs scipy
#include <BioFilterLib.h>

const float32_t notchCoeffs[5] = {
   0.99034f,  // b0
  -1.80193f,  // b1
   0.99034f,  // b2
   1.80193f,  // a1 (negado)
  -0.98068f   // a2 (negado)
};

// 1 etapa biquad, blockSize=1 (tiempo real)
IIRFilter notch60Hz(notchCoeffs, 1, 1);

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);
}

void loop() {
  float32_t raw     = (analogRead(A0) - 2048.0f) / 2048.0f;
  float32_t filtrada = notch60Hz.processSample(raw);

  Serial.print(raw,    6); Serial.print(",");
  Serial.println(filtrada, 6);
  delayMicroseconds(1042);
}

Comparativa FIR vs. IIR

CARACTERÍSTICA FIRFilter IIRFilter
Estabilidad ✓ Siempre estable ⚠ Puede ser inestable (coeficientes mal calculados)
Respuesta de Fase ✓ Lineal (simétrico) ⚠ No lineal — distorsiona morfología
Eficiencia (CPU/RAM) Moderada — más coeficientes ✓ Alta — pocos coeficientes
Retardo de Grupo $(N-1)/2$ muestras (constante) ✓ Mínimo (~0 para notch)
Buffer de Estado numTaps + blockSize - 1 floats ✓ Solo 4 × numStages floats
