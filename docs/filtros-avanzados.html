Filtros Avanzados: Wavelet y LMS

Para señales biomédicas no estacionarias —como el ECG con artefactos de movimiento o el EEG con parpadeos— se necesitan técnicas más sofisticadas. WaveletFilter descompone la señal en tiempo y frecuencia simultáneamente; LMSFilter se adapta en tiempo real al ruido cambiante.

WaveletFilter — Banco de Filtros Daubechies-4

ARQUITECTURA INTERNA — COMPOSICIÓN DE FIRFilter
ETAPA DE ANÁLISIS
*_approxFilter → Pasa-Bajo (DB-4)
*_detailFilter → Pasa-Alto (DB-4)
ETAPA DE SÍNTESIS
*_synthApproxFilter→ Reconstrucción
*_synthDetailFilter→ Reconstrucción

Cada uno de los 4 filtros internos es una instancia de FIRFilter, que a su vez encapsula arm_fir_instance_f32. La wavelet no usa CMSIS directamente; la implementa como un banco de 4 filtros FIR.

functions Fundamento Matemático

A diferencia de la Transformada de Fourier (que solo da información de frecuencia), la Transformada Wavelet Discreta (DWT) descompone la señal en componentes que tienen localización tanto en tiempo como en frecuencia. Se implementa como un banco de dos filtros:

FILTRO DE APROXIMACIÓN (PASA-BAJO)
$$A[n] = \sum_{k=0}^{7} h[k] \cdot x[n-k]$$

Extrae componentes de baja frecuencia. Representa la tendencia y forma base de la señal.

FILTRO DE DETALLE (PASA-ALTO)
$$D[n] = \sum_{k=0}^{7} g[k] \cdot x[n-k]$$

Extrae componentes de alta frecuencia. Representa bordes, transitorios y ruido.

La reconstrucción perfecta se obtiene combinando ambas salidas con los filtros de síntesis $\tilde{h}$ y $\tilde{g}$:

$$\hat{x}[n] = \sum_{k} \tilde{h}[k]\,A[n-k] \;+\; \sum_{k} \tilde{g}[k]\,D[n-k]$$

Wavelet Daubechies-4 (DB-4)

La librería usa coeficientes Daubechies de orden 4 (8 coeficientes por filtro). Son ideales para bioseñales porque:

check_circle
Reconstrucción perfecta: Los filtros de análisis y síntesis son ortogonales — no hay pérdida de información.
check_circle
Soporte compacto: Solo 8 coeficientes. Eficiente en CPU y RAM para el Cortex-M3.
check_circle
Fase casi lineal: Preserva mejor la morfología del QRS en ECG que otras wavelets.

Constructor y Métodos

Constructor 
WaveletFilter(uint16_t blockSize);
Los coeficientes DB-4 están predefinidos. Solo necesitas el blockSize: usa 1 para tiempo real.
processSample — análisis 
void processSample(float32_t input, float32_t* approxCoeff, float32_t* detailCoeff);
Aplica los filtros de análisis. Escribe el coeficiente de aproximación (baja frecuencia) y el de detalle (alta frecuencia).
reconstruct — síntesis 
float32_t reconstruct(float32_t approxCoeff, float32_t detailCoeff);
Combina aproximación + detalle para reconstruir. Truco de filtrado: para eliminar ruido de alta frecuencia, pasa approxCoeff y 0.0f como detalle.
reset 
void reset();
Limpia el estado interno de los 4 filtros FIR. Usar al cambiar de paciente o iniciar nueva sesión de adquisición.

lightbulb Ejemplo — Denoising de ECG con Wavelet

terminalWavelet_ECG_denoising.ino
// Denoising de ECG usando Wavelet DB-4
// Aproximación = señal suavizada (sin ruido de alta frecuencia)
// Detalle      = ruido + transitorios (picos QRS)
#include <BioFilterLib.h>

WaveletFilter ecgWavelet(1); // blockSize=1 para tiempo real

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);
}

void loop() {
  float32_t raw = (analogRead(A0) - 2048.0f) / 2048.0f;

  float32_t approx, detail;
  ecgWavelet.processSample(raw, &approx, &detail);

  // Señal denoised: reconstruir solo con aproximación
  float32_t denoised = ecgWavelet.reconstruct(approx, 0.0f);

  // Detección de QRS: el detalle tiene picos R
  if (abs(detail) > 0.15f) {
    // Posible complejo QRS detectado
    digitalWrite(LED_BUILTIN, HIGH);
  } else {
    digitalWrite(LED_BUILTIN, LOW);
  }

  // Visualizar las 4 señales en Serial Plotter
  Serial.print(raw,      4); Serial.print(",");
  Serial.print(approx,   4); Serial.print(",");
  Serial.print(detail,   4); Serial.print(",");
  Serial.println(denoised, 4);

  delayMicroseconds(1042); // 960 Hz
}

LMSFilter — Filtro Adaptativo LMS

CMSIS-DSP SUBYACENTE
arm_lms_norm_instance_f32

Encapsula arm_lms_norm_instance_f32 e invoca arm_lms_norm_f32(). Usa el algoritmo LMS Normalizado (NLMS), más estable que el LMS estándar porque normaliza el paso de adaptación por la potencia de la señal.

functions Fundamento Matemático — Algoritmo LMS

El LMS actualiza iterativamente sus pesos $\mathbf{w}[n]$ para minimizar el error cuadrático medio entre la señal de referencia $d[n]$ y la salida estimada $y[n]$:

① SALIDA DEL FILTRO
$$y[n] = \mathbf{w}^T[n]\,\mathbf{x}[n] = \sum_{k=0}^{M-1} w_k[n]\,x[n-k]$$
② SEÑAL DE ERROR
$$e[n] = d[n] - y[n]$$

Diferencia entre la señal deseada $d[n]$ (referencia) y la estimación del filtro. Decrece a medida que el filtro converge.

③ ACTUALIZACIÓN DE PESOS (NLMS)
$$\mathbf{w}[n+1] = \mathbf{w}[n] + \mu \cdot e[n] \cdot \frac{\mathbf{x}[n]}{\|\mathbf{x}[n]\|^2 + \epsilon}$$

$\mu$ es el paso de adaptación (step size). La normalización por $\|\mathbf{x}\|^2$ hace al NLMS más estable que el LMS clásico ante variaciones de potencia de la señal.

Selección del Paso de Adaptación $\mu$

El parámetro mu es el más crítico del filtro LMS. Controla el balance entre velocidad de convergencia y estabilidad:

SEÑAL MU RECOMENDADO TAPS TÍPICOS USO
ECG0.01 – 0.0532 – 64Cancelación 50/60 Hz, deriva de línea base
EMG0.001 – 0.0164 – 128Artefactos de movimiento, supresión de ruido
EEG0.0001 – 0.00132 – 64Cancelación de parpadeo (EOG como referencia)
info

¿Por qué usar LMS en vez de un Notch fijo?

Un filtro notch IIR elimina exactamente 60.0 Hz. Pero en la práctica, la frecuencia de la red puede variar entre 59.5–60.5 Hz según la carga, y la impedancia de los electrodos cambia durante la sesión. El LMS se adapta automáticamente a estas variaciones en tiempo real, sin necesidad de rediseñar el filtro.

Constructor y Métodos

Constructor 
LMSFilter(float32_t* coeffs, uint16_t numTaps, float32_t mu, uint16_t blockSize);
Los coeffs se inicializan típicamente a cero. Son modificados internamente durante la adaptación.
processSample 
void processSample(float32_t input, float32_t reference,
                      float32_t* output, float32_t* error);
input = señal contaminada · reference = señal de ruido puro (ej: seno de 60 Hz) · output = señal limpia · error = señal de convergencia.
setMu / getMu / resetCoefficients 
void      setMu(float32_t newMu);
float32_t getMu() const;
void      resetCoefficients(const float32_t* newCoeffs = nullptr);
setMu() ajusta la velocidad de adaptación en tiempo real. resetCoefficients() reinicia los pesos (útil si el filtro diverge).

lightbulb Ejemplo — Cancelación Adaptativa de 60 Hz en ECG

terminalLMS_cancelacion_60Hz.ino
// LMS: cancelación adaptativa de interferencia de 60 Hz en ECG
// La señal de referencia es un seno de 60 Hz generado localmente
#include <BioFilterLib.h>

const uint16_t NUM_TAPS = 64;
const float32_t MU      = 0.01f;   // Step size para ECG
const float32_t FS      = 960.0f;  // Hz de muestreo

float32_t lmsCoeffs[NUM_TAPS] = {0}; // Inicializar a cero
LMSFilter cancelador(lmsCoeffs, NUM_TAPS, MU, 1);

float tiempoSeg = 0.0f;

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);
}

void loop() {
  // Señal primaria: ECG + interferencia de red
  float32_t ecgContaminado = (analogRead(A0) - 2048.0f) / 2048.0f;

  // Señal de referencia: seno puro de 60 Hz
  float32_t ref60Hz = sin(2.0f * PI * 60.0f * tiempoSeg);

  float32_t ecgLimpio, errorAdaptacion;
  cancelador.processSample(ecgContaminado, ref60Hz,
                          &ecgLimpio, &errorAdaptacion);

  // Visualizar en Serial Plotter (4 señales)
  Serial.print(ecgContaminado,  4); Serial.print(",");
  Serial.print(ref60Hz,         4); Serial.print(",");
  Serial.print(ecgLimpio,       4); Serial.print(",");
  Serial.println(errorAdaptacion, 4);

  tiempoSeg += 1.0f / FS;
  delayMicroseconds(1042);
}
Interpretación de la señal de error: Si errorAdaptacion decrece con el tiempo → el filtro está convergiendo correctamente. Si oscila sin converger, reduce mu. Si es muy lento, auméntalo.
